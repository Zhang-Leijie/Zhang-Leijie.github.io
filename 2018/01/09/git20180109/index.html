Git基本使用方法

发表于 2018-01-09

Git是一个分布式版本控制系统，我们所熟悉的使用有github，gitlab等代码管理工具，都使用了git。但我们在使用这些工具时，可能较多的用到的是对应的GUI软件，但是这些软件具有很大的局限性，我们只有在命令模式下才能够执行Git的所有命令。Git总的来说是一个较为庞大的工程，在这里，我将以实用为主，主要来介绍一下Git的基本功能和我们常用到的一些命令。

Git是一个版本控制系统，具体来说就是一种记录一个或若干个文件内容变化，以便将来查阅特定版本修订情况的系统。

如今大多数人选择使用Git，Git对于其他类似的系统，总结了以下几个优点

  • 直接记录快照，而非差异比较，Git对待数据更像是一个快照流。如果文件没有修改，Git不再重新存储该文件，而是只保留一个链接指向之前存储的文件，这样做十分地高效。
  • 近乎所有操作都是本地执行。在Git中的绝大多数操作都只需要访问本地文件和资源，一般不需要来自网络上其它计算机的信息，这样使大部分的操作都是瞬时完成的。
  • 保证完整性。Git中所有数据在存储前都计算校验和，然后以校验和来引用。在很大程度上保证了数据的完整性和安全性。
  • 一般只添加数据。你执行的Git操作，几乎只往Git数据库中增加数据。很难让Git执行任何不可逆操作，或者让它以任何方式清除数据。这样很大程度上避免了
  • 文件具有三种状态：已提交（committed）、已修改（modified）和已暂存（staged），操作起来方便快捷

下面来介绍一下Git的一些基本操作：

在使用前我们可以先升级一下我们的Git:$ git clone git://git.kernel.org/pub/scm/git/git.git

如果相对Git的命令有一个概览，可以用这个方法:$ git help config

初始化仓库: $ git init

克隆现有的库: $ git clone [url]

检查当前文件状态: $ git status

跟踪新文件: $ git add [path]

提交更新: $ git commit

跳过使用暂存区域: $ git commit -a 可以对已经跟踪过的文件跳过 $ git add [path] 步骤

查看提交历史 $ git log

撤销操作: —amend

查看远程仓库: $ git remote -v

从远程仓库中拉取: $ git fetch [remote-name]

推送到远程仓库: $ git push [remote-name] [branch-name]

添加标签: $ git tag

Git的分支模型是其很强大的一个功能，对我们的版本管理发挥了很重要的作用，Git处理分支的方式十分轻量，创建新分支几乎可以在瞬间完成，不同分支之间的操作也十分便捷。

创建分支: $ git branch [name]

切换分支: $ git checkout [name]

合并分支: $ git merge [name]

删除本地分支: $ git branch -d [name]

删除远程分支: $ git push origin —delete [name]

获取分支列表: $ git branch

获得远程分支信息: $ git remote show

变基: $ git rebase [name] 结果与使用merge没有区别，但是变基使提交历史显得更加简洁。

分支开发作为很重要的一个功能，也有着不同的使用方式：

长期分支：在整个项目开发周期的不同阶段，你可以同时拥有多个开放的分支；你可以定期地把某些特性分支合并入其他分支中。

特性分支：特性分支对任何规模的项目都适用。 特性分支是一种短期分支，它被用来实现单一特性或其相关工作。

上面只是简单地列举了一些Git的基本操作，读者如果想熟练地掌握这些操作，还是要自己去创建一个项目，实际的运用这些命令。这些命令也只是Git的冰山一角，如果想要有更多的了解，可以参考 Pro Git 这本书。
