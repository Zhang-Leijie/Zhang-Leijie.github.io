mostjs源码阅读一

发表于 2018-01-06

mostjs是实现RP技术的一个不错的库，在其官方的github库上对其有这样的描述：It features an ultra-high performance, low overhead architecture, APIs for easily creating event streams from existing sources, like DOM events, and a small but powerful set of operations for merging, filtering, transforming, and reducing event streams and time-varying values.。对于其中的ultra-high performance的描述，我有着浓厚的兴趣，这次的源码阅读以此为切入点，想看看mostjs是使用了什么魔力，使其可以以很高的性能来实现较为复杂的RP技术。

首先，我们来到的第一个方法是most.of(x):

mostjs 通过这三个简单的函数构造了一个初步的Stream，相当于创建了一个{ source: { value: x } }的对象，可以看到，其构建的方法十分简单，所以速度相当之快。

我们接下去来到了observe()这个函数

从这里我们可以看出Stream.prototype.observe，实际上是对先前生成的Strean做的一个遍历操作，同时在这里加入了一个回调函数f,对于我们所使用的事例代码来说，其实就是function (val) { x = val }。

从这几个函数里，mostjs对传入的Stream又进行了一次包装，并且把函数f包裹在了其中

这里第一次出现了 run 这个关键词 说明到这里对生成Stream这一步已经基本完成，下面就是对这个Stream进行的各种操作了。

这里要做一个说明 此处出现的withDefaultScheduler在observe中叫run，承接上面一个函数的run(stream.source)。在withScheduler中出现了defaultScheduler这个参数，这个参数在之前并没有遇到过，我们可以跳转到defaultScheduler，在此处增加一个debug，来看看这个参数是怎么来创建的。

我们先简单的来看这几个函数，Scheduler 是mostjs里面核心的函数之一，需要传入两个参数，分别是一个时钟和一个列表，初步来看，时钟是一个空函数，任务列表是一个数组，都是代码中十分简单的构成。在这里也可以从侧面反映出mostjs的代码执行效率是很高的，所以他的速度很快。defaultScheduler这个参数就是这样生成的，我们再回到刚刚的步骤上来。

这里出现了另一个mostjs中比较核心的disposable。

通过这两个函数创建了一个disposable。同时这里也定义了一个observer，是一个Drain，同时Drain也定义了许多属性。接下去便是source.run(observer, scheduler)，这里的source对应的是Tap生成的。

我们在这里又看到了一个source.run,我们可以回忆一下，这里对应的应该是Just.prototype.run，我们要注意的是在这里不要把各个参数，方法给搞混了。

在这里又回到Scheduler这个函数，这个函数是操作task的关键，接下去会以此为展开介绍task的生命流程

下面我们来到了这个过程的结束阶段，下面我们会执行sink.end(t, void 0)。

这这里会执行一步disposeThen(this._end, this._error, this._disposable, x)，dispose的的概念在mostjs中是比较复杂和关键的，我打算单独来分析一下这个概念，在这里就不再很详细的展开了，在这里我们需要了解的是通过这个函数，我们会在原有的tasks数组中移除已经执行过的task。接下去我们会回到runTask函数中的那个循环中，来执行接下去的task。当我们没有多余的task后，我们会通过dispose回到最外层。
